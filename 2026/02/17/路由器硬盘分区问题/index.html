路由器 eMMC 全盘分区详解

以 8GB eMMC 路由器 GPT 分区为例
适用架构：Qualcomm IPQ 系列
系统类型：OpenWrt / Kwrt
存储类型：eMMC 8GB（实际约 7.22 GiB）
分区表类型：GPT（2048 扇区 / 1MB 对齐）

整体空间结构划分

整块 eMMC 可逻辑划分为三大区域：

  1. Boot 引导区（p1–p17）
  2. 系统核心区（p18–p25）
  3. 用户可用区（p26）

Boot 引导区（p1–p17）

启动链原理

Qualcomm 平台采用分级加载启动机制：

  • BootROM 固化在芯片内部，不可修改。
  • eMMC 中的 p1–p17 承载外部启动组件。
  • 任一关键分区损坏，将导致启动链中断。
  • 结果通常为：无串口输出、无法软件恢复。

区域特性说明

  • 总容量约 20MB
  • 由厂商出厂写入
  • GPT 仅描述分区位置，不代表可操作
  • 严禁删除、格式化或改动大小
  • 若损坏，仅能通过 EDL / 专用烧录工具恢复

用户可用区（p26,p27）

这是唯一可安全操作的分区。

分区 起始-结束 大小 建议用途
p26 5212160-15146974 4.74 GB 数据存储

分区对齐间隙

产生原因

GPT 分区默认按 2048 扇区（1MB）对齐。

原因：

  • 提升 eMMC 读写性能
  • 避免跨擦除块写入
  • 适配闪存页结构

对齐间隙明细

这些空间不可利用.
