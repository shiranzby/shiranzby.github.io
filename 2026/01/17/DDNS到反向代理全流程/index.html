路由器DDNS到域名访问服务全流程

分为两种情况

只需要HTTP基础访问功能
需要HTTPS加密访问

只需要HTTP访问（非加密）

注册免费域名

免费域名注册

默认可免费注册三个

如果只需要HTTP访问则需要拿到API

API要求有五个免费域名

需要要求两个好友才能拿到五个

记录下API Key和API Secret

Openwrt安装动态DNS

在系统 软件包 查找luci-app-ddns、luci-i18n-ddns-zh-cn安装

安装好后刷新页面

登录到路由器的 SSH（或使用 TTYD 终端），输入下面命令一键部署脚本

配置 LuCI 界面

回到 OpenWrt 界面（服务项 » 动态DNS » myddns_ipv6），按照以下方式填空：

基础设置

  • 勾选已启用
  • 域名: yourhost.example.com (填写你的实际完整域名，必须要在你的 API 账号里已经存在)
  • IP 地址版本: IPv6
  • DDNS 服务提供商 [IPv4/IPv6]: 选择 自定义
  • 自定义更新 URL: (留空，不要填)
  • 自定义更新脚本: /etc/ddns/update_dnshe_ipv6.sh
  • 用户名: 填写你的 API Key
  • 密码: 填写你的 API Secret
  • 使用 HTTPS: (如果脚本不工作，可以尝试取消勾选，但在脚本模式下这个选项通常只影响 wget/curl 库的选择，脚本里我们强制用了 curl，所以这个选项影响不大，建议勾选)
  • CA证书路径: 默认留空，会自动填入IGNORE

高级设置

  • IP地址来源：网络
  • 网络：wan6（有ipv6的）
  • 勾选记录日志到文件
  • 其余默认

定时器设定

  • 检查间隔：10
  • 检查间隔单位：分钟
  • 强制更新间隔：3
  • 强制更新间隔单位：天
  • 其余默认

防火墙区域设置

常规设置全部接受

下方区域全部接受

通信规则页可再配置WAN到本设备接受入站、WAN到LAN接受转发

注意，这样谁都可以访问进入你的局域网，极其危险！！！

如需精细化管理，可单独只开部分端口

补充说明

以上配置好后，就可以通过域名加端口形式访问路由器部署的服务

如果像访问内部局域网设备，则需要配置相应端口转发

家庭宽带默认不开放80和443端口

由于是非加密的访问，建议开放的服务都配置账号密码

需要HTTPS加密访问，托管到Cloudflare并部署证书

域名托管到Cloudflare

Cloudflare官网

进入后登录

填写要托管的域名

下一步

如果可以选择套餐了就说明可以托管

接着下一步

会出现两个NS用的服务器

记录下来

点左侧SSL/TLS Origin Server

创建证书

记录两串字符

上面的保存为 .pem后缀文件

下面的保存为 .key后缀文件

接着获取全局密钥

全局密钥在Cloudflare右上角个人中心

左侧API令牌

下方API密钥Global API KEY

需要收邮箱验证码查看

保留好全局密钥

DNSHE网页端配置

找到要托管的域名

点击对应域名的右侧DNS服务器

设置NS服务器列表

填入Cloudflare给的两个Cloudflare NameServers

相应每行填入一个

勾选强制替换

保存后如下所示

Openwrt端配置

路由器安装ddns配套cloudflare插件

Lucky端配置

自行安装Lucky

进入后禁用安全入口设置检查

设置好用户名密码

打开防火墙自动放行

下面保存

点击左侧SSL/TLS证书

添加证书

证书备注随意

添加方式选文件

证书一栏上传前面保存的.pem后缀的文件

Key一栏上传前面保存的.key后缀的文件

点击添加即可

点击左侧Web服务

添加Web服务规则

规则名称随意

规则开关启用

操作模式 简易模式

监听类型必须打开IPv6，IPv4随意

监听端口填想通过域名加端口形式访问的端口

打开防火墙自动放行

启用TLS 版本限制TLS1.2

默认规则部分

默认标记关

服务类型反向代理

默认目标地址 127.0.0.1:端口号

该端口号为服务部署实际占用端口，即想代理出去的服务端口号

打开万事大吉

忽略后端TLS证书验证

打开自动反代重定向

默认记录访问日志

点击修改WEB服务规则

此时外网就可以通过HTTPS安全访问了
