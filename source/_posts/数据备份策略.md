# 最终备份方案

## 🧭 备份策略总览

为了确保个人数据的**安全性、可恢复性与使用便捷性**，本方案制定如下总目标：

### 🎯 目标原则

1. ✅ **Free 优先 / 极力节省成本**
   - 优先使用免费工具与服务（如夸克网盘、FreeFileSync、GitHub 私有仓库等）。
   - 如遇容量/功能限制，可在关键场景下考虑“轻量付费”。
2. 🧊🔥 **冷备份 + 热备份并存**
   - **冷备份**（如外置硬盘、月度虚拟机镜像）：面向大规模恢复与长期留存。
   - **热备份**（如夸克同步文件夹、WebDAV）：用于日常使用/误删恢复。
3. ✌️ **双重备份**
   - 所有重要数据均至少存在两份：本地一份 + 云端一份。
   - 特别关键数据可建立“三重冗余”（如代码库、密码数据库）。
4. 🔐 **安全、便携、实用**
   - 所有备份路径可在新设备/系统中快速恢复（不依赖某一单一设备）。
   - 敏感文件使用 `.gitignore` / 加密等方式保障隐私。
   - 所用工具应支持主流系统（Windows/macOS/Linux/手机）。
5. 🌀 **热备尽量做到“无感备份”，自动化脚本**
   - 使用同步客户端（夸克、天翼）或脚本（FreeFileSync + 定时任务）实现定时自动化。
   - 避免手动操作，提高容错率。
6. 📖 **编写《恢复手册.md》**
   - 文档中将详细记录：
     - 所有数据来源位置（路径 / 网盘）
     - 对应恢复方式与工具
     - 账号密码管理器路径
     - 依赖的插件/脚本清单
   - 保证任何时候数据在 24 小时内可通过登录账户 + 同步脚本 + 手动恢复操作，实现完整数据还原。

#### 🧐 为什么选择 Google Photos + 一刻相册作为主力云端备份方案？

##### ✅ 1. Google Photos（谷歌相册）

- 我已搭建软路由，家中设备默认处于“魔法环境”，**Google Photos 可实现无感自动同步**。

- **优点**：
  - **无限容量优势**：Google 官方承诺 **Pixel 1 机型享有永久、无损、无限量照片与视频备份**。
  - **技术绕过可实现“伪Pixel”特权**：通过 Root 的手机设备 + Magisk 模块，可模拟为 Pixel 1，激活无限备份策略。
  - **自动分类、搜索强大**：Google Photos 提供 **AI 智能识别、人脸聚合、地图定位、关键词搜索** 等强大功能，方便管理海量照片。
- **缺点**：
  - Google 在中国大陆无法直接访问，**必须依赖稳定 VPN 或旁路代理环境**。

##### ✅ 2. 一刻相册（作为国产补充方案）

- **优点**：
  - 理论无限空间、界面友好。
  - 适合中国用户使用。
- **缺点**：
  - 备份大视频、批量恢复需开通会员。
  - 卡顿，使用体验并不好，自动备份实测耗电情况严重。

##### ✅ 3. 本地备份（未来补充方案）

- 本地方案，计划后续部署 **NAS + 易有云** 搭建家庭私有云，实现本地备份、快速访问和远程同步。
- 配合 Google Photos 可形成「冷 + 热备份」组合，提高数据安全性和可恢复性。

### 🎬 影视

#### ☁️ 云端备份策略

- **115 网盘**（主力推荐）
  - ✅ 稳定性高，上传/下载/播放速度极佳。
  - ✅ 支持 WebDAV 协议，可被 Jellyfin、Infuse、nPlayer 等播放器远程挂载使用。
  - ✅ 支持在线播放，大视频兼容性强。
  - ⚠️ **需付费会员才能正常使用**，非会员限速严重，功能大幅受限。
  - ⚠️ 分享功能有限制，建议搭配 AList 或其他工具使用。
- **天翼云盘**（适合电信用户）
  - ✅ **永久免费扩容活动多**（绑定手机号后空间可达 6TB+）。
  - ✅ 上传/下载速度快，支持自动备份监控文件夹。
  - ✅ 适合作为长期备份盘，配合自动化工具使用更佳。
  - ⚠️ 播放器兼容性一般，不支持 WebDAV，适合作为存储盘而非播放源。
- **移动云盘**（适合移动用户）
  - ✅ 与天翼云盘类似，用户活跃度高时也有较大空间赠送。
  - ✅ 上传/下载体验不错，适合作为副备份盘。
  - ⚠️ 播放体验一般，不支持 WebDAV。
- **夸克网盘**（次级推荐）
  - ✅ 支持在线播放、AI字幕识别、自动分类等功能。
  - ✅ 可通过淘宝88VIP（88元/年）获得夸克会员，**相当于“白嫖”会员体验**。
  - ⚠️ 免费用户限速严重，长时间使用必须配合会员。
  - ⚠️ 分享和下载频率受限，适合当临时备份盘。
- **123 网盘**（备用方案）
  - ✅ 上传/下载/在线播放体验优秀，客户端简洁无广告。
  - ✅ 分享功能较开放，支持大文件外链。
  - ✅ 官方有 Web API 支持，可接入 AList 或自建下载脚本。
  - ⚠️ **无会员使用体验较差**，限速较严重，上传大文件基本不可行。
  - ⚠️ 存储安全性有待考察，**适合短期转存或资源中转**。

#### 🧩 聚合播放方案：小雅 AList

- AList 是聚合网盘的 Web 文件浏览器，可统一访问多个云盘资源。
- 支持网盘包括：**阿里云盘、115、123、坚果云、天翼云盘（部分支持）、WebDAV、自建服务等**。
- 可配置为 WebDAV 网关，与 Jellyfin、Infuse、Kodi、nPlayer 等播放器联动使用，实现局域网或远程无感播放。
- 推荐搭配方案：
  - 小雅 AList 公共节点（alist.xiaoya.pro）→ 挂载 115 / 123 网盘 → Infuse / nPlayer 播放器观看。
  - 自建 AList 实例（Docker / 本地）+ Token 授权更安全稳定。

### 音乐

**本地**：当前占用约10GB，统一存放于固定路径（`E:/音乐/`），冷备份至U盘及软路由存储中，确保本地多设备冗余。

- **本地播放/管理方案**：采用 **Navidrome + NAS** 组合或 **Jellyfin + NAS**，搭建私人音乐服务器，实现跨设备在线播放、管理和元数据同步。
  - **Navidrome**：轻量级音乐服务器，支持多用户、Web和手机客户端，兼容多种音频格式，适合个人或小团队使用。
  - **Jellyfin**：功能更强大，支持多媒体统一管理（音乐、电影、电视），支持DLNA、Chromecast等多种播放协议，适合更丰富的多媒体需求。

**云端同步**：

- 夸克网盘：启用“同步文件夹”功能，将`D:/Music/`设为同步目录热备份（需确保夸克客户端常驻后台）。

**歌单与播放列表管理**：

- 使用 **歌单无界** 等工具，能有效匹配和同步歌单排序，实现云端与本地播放列表的无缝衔接，避免歌单乱序问题，提升听歌体验。

- 支持 **Android / iOS / Web / Windows / Mac** 等主流平台。

- **性价比极高**，几乎免费获得专业级云盘体验。

##### ✅ 上传/下载速度优秀

##### ✅ 原文件存储，不转码不压缩

- 音频文件 **不会被转码或重命名**，保留完整元数据。
- 适合储存音乐资料库、专辑封面、歌词、Cue 文件等文件结构较复杂的音乐归档。

##### ✅ 文件结构清晰，支持批量管理

- 支持自定义目录层级，如 `音乐归档/无损合集/年代/艺术家/`，便于分类和查找。
- PC客户端支持拖拽和批量上传，简化管理流程。

### 代码工作区（`F:/Workspace/`）

#### 本地备份

##### 当前状态

- 主要代码工作区位于F盘本地磁盘。

##### 未来规划

- 计划部署 **NAS 设备**，实现自动定时快照和备份。
- NAS结合版本控制或增量备份方案，可大幅提升数据安全性和多设备访问便利。

#### 云端同步

##### Git + GitHub 私有仓库（适合核心代码版本控制）

###### 注册与创建仓库

- 注册GitHub账号（https://github.com/）。
- 创建私有仓库（例如 `my-workspace`）。
- **创建时避免勾选“Initialize with README”**，防止初始化文件与本地冲突。

###### 初始化本地仓库并关联远程

- 在目标项目目录（如`F:/Workspace/project1/`）打开 **Git Bash Here**，依次执行：

  ```bash
  git init
  git remote add origin https://github.com/你的用户名/my-workspace.git
  git add .
  git commit -m "Initial commit"
  git push -u origin main
  ```

###### 日常更新流程

- 每次代码修改完成后，执行：

  ```bash
  git add .
  git commit -m "简短描述本次修改"
  git push
  ```

###### 多设备恢复

- 在新设备或工作环境，执行克隆操作：

  ```bash
  git clone https://github.com/你的用户名/my-workspace.git
  ```

###### 建议辅助工具

- 推荐使用 VSCode 等支持Git的编辑器，内置Git插件使操作更直观便捷。

###### 注意事项

- 避免提交大文件或敏感信息。
- 可通过 `.gitignore` 文件排除不需要同步的文件。

##### 夸克网盘（适合非版本控制的备份）

###### 适用范围

- 备份无需版本控制的代码副本、资料文件、配置文件或二进制资源。

###### 备份方法

- 在夸克网盘创建同步文件夹（如 `Workspace_Backup`）。
- 将本地目录 `F:/Workspace/` 拖入同步文件夹，实现自动热备份。
- 保证夸克客户端常驻后台以确保同步稳定及时。

#### 补充建议

##### 自动备份脚本

- 可编写定时任务脚本，自动复制代码工作区至NAS或备份目录。
- 结合增量备份工具，如Linux的 `rsync`，Windows下推荐使用 `DeltaCopy`，提高备份效率。

##### 多版本备份策略

- 利用Git的版本管理优势管理代码历史。
- 结合NAS快照功能，实现数据误删、误覆盖的快速恢复。

##### 安全性建议

- 私有仓库不宜存储明文密码或敏感密钥。
- 建议使用环境变量或专门的安全管理工具处理敏感信息。

### QQ/微信聊天记录

#### 手机 → 电脑备份

##### 微信备份

- 通过电脑版微信“备份与恢复”功能，连接手机和电脑，选择“备份聊天记录至电脑”操作（需同一局域网）。
- 可备份聊天文本、图片、语音等内容，需手动操作。

##### QQ备份

- 进入手机QQ“设置 → 通用 → 聊天记录备份与恢复”，选择“备份聊天记录到电脑QQ”，手机与电脑需联网且在同一网络。

#### 电脑 → 本地存储

##### 默认存储路径

- 微信聊天记录路径为：
   `C:/Users/<用户名>/Documents/WeChat Files/<微信号>/`
- QQ聊天记录路径为：
   `C:/Users/<用户名>/Documents/Tencent Files/<QQ号>/`

##### 备份文件管理

- 建议定期将聊天记录目录压缩成 `DataBackup.zip` 方便管理和迁移。
- 压缩包可存储于U盘、NAS或上传至云盘（如夸克网盘）实现多重备份。

#### Root手机完整备份（推荐第三方工具“数据备份”）

##### 为什么推荐“数据备份”

- 经多次测试，手机官方微信和QQ备份功能**容易遗漏大量图片和视频**，导致备份不完整。
- “数据备份”作为Root权限工具，能**完整导出所有QQ和微信数据**，包括聊天记录、图片、视频、数据库和缓存文件。

##### 备份流程

- 在Root的手机设备上运行“数据备份”软件，进行完整数据导出。
- 备份数据压缩成 `DataBackup.zip` 文件，方便保存与传输。
- 上传压缩文件到夸克网盘或其他云盘，实现安全、长期存储。

##### 优势说明

- 该备份方式全面且数据完整，适合设备更换、系统重装时恢复使用。
- 极大减少因官方备份功能不足造成的数据丢失风险。

#### 自动化备份建议

##### Windows定时压缩

- 利用任务计划结合批处理脚本，定期压缩本地聊天记录文件夹，减轻手动备份负担。

##### 多层备份架构

- 结合软路由与NAS，定期同步聊天数据，实现本地与网络的多层冗余备份。

#### 安全与隐私

##### 数据加密

- 聊天记录包含个人隐私，建议使用7-Zip等工具对压缩包设置密码保护。

##### 云盘权限控制

- 上传云盘后，务必设置访问权限，避免数据泄露风险。

##### Root权限风险

- Root操作存在一定风险，务必保证备份工具来源正规可靠。

#### 恢复方法

##### 官方恢复途径

- 电脑版微信和QQ支持“备份与恢复”功能，可恢复聊天记录（需确保备份文件完整且版本匹配）。

##### Root备份恢复

- 需借助相应的手机恢复工具，将完整备份数据导入手机，恢复聊天内容、图片和视频。

##### 手动恢复

- 关闭对应客户端后，手动复制聊天记录文件夹到原路径，实现基础数据恢复。

### 浏览器书签（Chrome）

- **本地备份**：可通过 Chrome 自带的导出功能，定期导出为 `.html` 文件保存到本地。
- **云端同步**：
  - 使用 **Floccus 浏览器插件**，配置 WebDAV（推荐坚果云）后，可实现自动同步浏览器书签到云端。
  - 也可手动导出书签 `.html` 文件，存入夸克网盘等云服务。

#### 🧐 为什么选择坚果云作为同步平台？

##### ✅ 完全支持 WebDAV 协议

- 坚果云提供长期稳定的 WebDAV 接口。
- 与 Floccus 等插件完美兼容，无需额外脚本或服务端配置。

##### ✅ 安装简便、自动同步

- 配置一次即可后台自动同步书签文件，无需手动导出。
- 支持多浏览器、多设备自动保持书签一致。

##### ✅ 空间充足，传输速度快

- 免费版即提供 1GB 存储，书签数据占用极小。

### 🛠 自动化备份与脚本整合

#### 🔄 自动化备份流程

为了实现备份的“无人值守、自动执行”，以下是推荐的自动化流程与工具：

##### ✅ 使用 FreeFileSync + Windows 任务计划程序 实现本地同步

1. 安装 FreeFileSync

2. 新建“同步任务”（Batch Job），每个任务可指定如下内容：

   - 源文件夹 → 目标文件夹（如 `F:/Workspace/` → `夸克同步文件夹/Workspace/`）
   - 选择同步方式：**镜像同步（Mirror）** 或 **更新同步（Update）**
   - 排除无关内容（如 `.git/`、临时文件夹、系统隐藏目录）
   - 勾选：同步后发送通知 / 日志记录
   - 保存为 `.ffs_batch` 文件

3. 打开 Windows“任务计划程序”：

   - 新建基本任务 → 设置触发时间（如“每天 9:00”）

   - 动作选择“启动程序”，指向 FreeFileSync 的可执行文件，并附上参数：

     ```
     FreeFileSync.exe "C:\路径\to\sync_task.ffs_batch"
     ```

4. 重复以上步骤，可对多个同步项目（音乐、代码、文档）设定不同计划任务。

5. 可勾选“开机自启 + 错过任务立即执行”选项，保障离线时也不会丢同步任务。

##### ✅ 可选增强工具：Rclone + 脚本

- 对于不支持同步客户端的云平台（如坚果云 WebDAV），可使用 Rclone 工具进行自动同步：

1. 下载并配置 Rclone：https://rclone.org/downloads/

2. 执行配置命令：

   ```
   rclone config
   ```

   - 输入 `n` 新建配置 → 输入名称（如 `jianguo`）
   - 类型选择 `WebDAV`
   - 填入坚果云信息：地址、用户名（邮箱）、密码（应用授权码）

3. 编写自动同步脚本 `saltnote_sync.bat`：

   ```
   @echo off
   rclone sync "C:\Users\你的用户名\Documents\SaltNote" jianguo:/saltnote/ --update --copy-links --log-file=sync.log
   ```

4. 将脚本添加至“任务计划程序”，设定定时执行。

5. 亦可设置为“系统启动时执行”。
